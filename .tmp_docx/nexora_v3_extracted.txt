1. VISION & POSITIONNEMENT PRODUIT
1.1 Définition
NEXORA est une plateforme SaaS de gestion intelligente dédiée aux entreprises de distribution de boissons. Elle couvre l'intégralité de la chaîne opérationnelle, de l'achat fournisseur à l'analyse financière, en passant par la logistique terrain africaine, la consignation et la gestion des créances.
Flux couvert : Achat → Stock → Commande → Terrain → Préparation → Livraison → Consignation → Encaissement → Analyse

2.3 Multi-tenancy
Architecture : Single Database, Row-Level Security via colonne tenant_id sur toutes les tables métier.
Middleware TenantResolver : identifie le tenant via sous-domaine ou header X-Tenant-ID
Scope Eloquent global automatique : toutes les queries filtrées par tenant_id
Isolation totale des données entre entreprises clientes
Configuration par tenant : logo, couleurs, paramètres métier

4.2 Module Consignation (Critique)
Élément différenciateur majeur. Toute livraison de boissons implique un dépôt de consignes (emballages) récupérées lors de la visite suivante.
Cycle de vie d'une consigne
Livraison : X casiers livrés → X consignes déposées chez le client
Visite suivante : consignes vides récupérées par le livreur
Retour dépôt : consignes entrées en stock emballages

4.3 Module Commercial Terrain (Ultra-Critique)
Contrainte offline — Réalité africaine
Le commercial opère en zone peu ou pas couverte. L'application doit fonctionner 100% offline et synchroniser automatiquement dès retour réseau.
Architecture offline — PWA + IndexedDB
Service Worker : cache des assets et API responses
IndexedDB : stockage local des commandes, clients, catalogue
Sync background : synchronisation via Background Sync API au retour réseau
Conflict resolution : last-write-wins avec horodatage serveur
